# 動畫迭代與物理更新邏輯詳解

本模擬器透過 `requestAnimationFrame` 建立高效且省電的動畫循環。以下是每一幀（Frame）的迭代詳解：

---

## 1. 動畫主迴圈 (`animate`)

這是模擬器的導航員，負責決定當前的狀態與是否繼續渲染：

1.  **更新馬達角度 (`theta`)**：
    - 若 `isPlaying` 為真，則根據 `simSpeed` 增加角度。
    - 同步更新 HTML 中的 `Manual Angle` 滑桿與數值顯示。
2.  **執行渲染與物理 (`renderFrame`)**：
    - 呼叫核心函數（見下節）。
3.  **自動休眠判定 (Auto-Sleep)**：
    - 檢查是否處於播放狀態，且物理系統是否仍有速度（`angularVel`）或偏差。
    - **休眠**：若馬達停止且物理已平衡，則停止 `requestAnimationFrame` 以節省資源。
    - **喚醒**：當使用者操作 UI 時，透過 `requestUpdate()` 重新啟動循環。

---

## 2. 渲染與物理核心 (`renderFrame`)

每一幀最繁重的計算都在這裡進行，分為四個階段：

### A. 運動學計算 (Kinematics)
- 根據目前的 `theta` 求解「三連桿機構」的幾何解。
- 透過 `getIntersection` (圓與圓交點) 找出各關節位置。
- 利用 `extendPoint` 算出前、中、後三隻腳的足端位置。

### B. 目標地面判定 (Ground Detection)
- 在「機體座標系」中，找出能支撐住重心（$x=0$）且讓所有腳都在線上的理想支撐線。
- 若有多組組合，選擇符合外凸包性質的一組。
- 此目標定義為 `targetM` (目標斜率) 與 `targetC` (目標高度)。

### C. 力矩物理更新 (Torque-Based Physics)
這是控制機器人「傾倒感」的核心：
1.  **支點選擇 (Pivot Selection)**：根據目標斜率與當前斜率的差異，判定機器人正往哪邊倒，並選出該側最邊緣的腳作為**旋轉支點**。
2.  **角加速度**：計算當前斜率與目標的差距，乘以 `gravityScale` 得到角加速度。
3.  **幾何約束旋轉**：
    - 更新斜率 `smoothedGround.m`。
    - **關鍵點**：截距 `c` 不再獨立積分，而是透過公式 $c = y_{pivot} - m \cdot x_{pivot}$ 強制推導。這確保了旋轉過程中支點永遠「釘」在地上。
4.  **碰撞攔截**：檢查是否超過目標值，若超過則直接「Snap」（鎖定）到目標並結束運動速度。

### D. 畫布渲染 (Drawing)
- **座標轉換 (`mapCoords`)**：
    - **Robot POV**：機體不動，背景地線旋轉。
    - **World POV**：地線不動（畫布底部的水平線），機體根據 `smoothedGround` 的參數進行相反方向的旋轉與垂直垂直位移補償。
- **繪圖順序**：路徑軌跡 $\to$ 地平線 $\to$ 機體主樑 $\to$ 連桿組 $\to$ 關節點。

---

## 3. 物理狀態總結

| 狀態名稱     | 行為描述                               | 休眠判定              |
| :----------- | :------------------------------------- | :-------------------- |
| **動態平衡** | 腳在走動，地面隨之平穩更新             | 否 (Playing = true)   |
| **自由傾倒** | 重心移位，機體因力矩加速旋轉           | 否 (Vel > 0)          |
| **穩定鎖定** | 碰到地面線，速度歸零，死轉過後完全靜止 | 是 (平衡後進入 Sleep) |
