# 蹦跳機械模擬方案總結

本文檔詳細介紹了「蹦跳機械」2D 機器人模擬器的主要程式碼實現策略與技術架構。

## 一、 視覺渲染策略

模擬器採用 **SVG (Scalable Vector Graphics)** 進行渲染，確保了在不同縮放級別下的清晰度，並利用以下技術提升視覺質感：

- **線性漸變 (Linear Gradients)**：為機械零件（如黃色主板、木質腿部、PVC 圓環）添加光影效果，使其具有立體感。
- **濾鏡效果 (Dropshadow Filter)**：為機械組件添加陰影，增強層次感與懸浮感。
- **樣式系統**：結合 HTML/CSS 提供響應式的容器界面，展現現代化的 Web App 風格。

## 二、 機械結構組件

機器人由多個各司其職的次系統組成：

1.  **左側圓環 (Left Ring)**：模擬 PVC 材質的輕量化平衡環。
2.  **主體結構 (Main Body)**：鮮艷的黃色橫向板材，承載電池盒與吊車機構。
3.  **電池盒 (Battery Box)**：位於後方，模擬配重塊以平衡整體重心。
4.  **懸掛吊車機構 (Crane Mechanism)**：核心動態組件，包含：
    -   **支架 (Bracket)**：固定於主板頂部。
    -   **金屬旋臂 (Blue Arm)**：負責高速旋轉產生離心力。
    -   **木質菱形配重 (Wooden Diamond)**：安裝於旋臂末端。
5.  **右側行走腿 (Right Leg)**：
    -   **大腿塊 (Thigh)**：垂直連接於主體。
    -   **小腿件 (Calf)**：透過數學計算的旋轉偏移量，確保其在靜態狀態下與地面完美貼合。

## 三、 動態模擬與動畫

模擬器不依賴 CSS 動畫，而是使用 **`requestAnimationFrame`** 進行精確的影格控制：

-  **持續旋轉**：吊車機構圍繞中心樞軸持續旋轉，角度隨 `deltaTime`（時間增量）與用戶輸入的 `speedMultiplier`（速加倍率）動態更新。
-  **平滑變速**：透過滑塊控制轉速，實現從 0.2x 到 5.0x 的平滑過度，而不重置旋轉狀態。

## 四、 重心 (COM) 計算邏輯

這是本模擬器最具技術性的部分，系統實時計算整體重心位置並更新 UI 上的 COM 標記：

1.  **密度模型**：為不同材料定義相對密度（木材: 0.6, PVC: 1.3, 金屬: 15.0, 電池: 4.0）。
2.  **質量加權平均**：
    -   **靜態質量**：基準板、圓環、大腿、小腿的質量與位置是固定的。
    -   **動態質量**：吊車旋臂與配重菱形的位置隨旋轉角度 $\theta$ (cos/sin) 實時變化。
3.  **公式實現**：
    $$TotalMass = \sum Mass_i$$
    $$COM_{x,y} = \frac{\sum (Position_{i} \times Mass_i)}{TotalMass}$$
4.  **實時視覺化**：粉紅色的 COM 標記隨吊車旋轉而小幅偏移，模擬真實機械運動中的重心抖動現象。
