# File Analyzer

這個技能能幫助你快速理解任何檔案的核心內容。當你在聊天中提到一個檔案（例如使用 `@filename`）或明確要求分析某個檔案時，我會執行深度讀取並提供結構化的分析。

## 觸發情境

當使用者執行以下操作時，請務必使用此技能：
- 在對話中使用了 `@` 標記特定檔案（如 `@simulation.js`）。
- 詢問「這檔案在幹嘛？」、「摘要一下這個檔案」、「分析這個檔案的邏輯」。
- 要求進行 Code Review 或尋找潛在問題。

## 分析流程

1.  **讀取檔案**：使用 `view_file` 讀取完整內容。
2.  **深度理解**：分析代碼邏輯、架構、依賴關係及邊界處理。
3.  **產出報告**：按照以下結構輸出。

## 輸出結構

請始終使用以下格式進行回覆：

### 🎯 核心主旨
> [用一句話描述這個檔案的根本存在目的]

### 💡 關鍵重點
- **[範疇 A]**: [具體描述該範疇的功能或邏輯]
- **[範疇 B]**: [具體描述該範疇的功能或邏輯]
- [更多重點...]

### 🔗 依賴關係
- **內部依賴**: [提及本專案中的其他檔案或模組]
- **外部依賴**: [使用的庫、API 或框架]

### ⚠️ 潛在風險 / 邊界情況
- [描述可能的 Bug、效能瓶頸或未考慮到的輸入情況]

### 🧐 專業評論
- [從資深工程師的角度評價該檔案的設計模式、可讀性或架構]

### 🛠️ 重構建議
- [給出具體、可執行的改進建議]

---

## 注意事項
- **保持精煉**：避免廢話，直接切入核心。
- **準確性**：確保分析與代碼事實相符，不要腦補。
- **繁體中文**：所有分析與說明均使用繁體中文。
