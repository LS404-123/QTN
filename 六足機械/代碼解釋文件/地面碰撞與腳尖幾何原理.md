# 橢圓腳部地面碰撞幾何原理

本文件詳細說明模擬器（`simulation.js`）如何動態計算橢圓形腳部與地面的接觸點（Contact Point）以及如何自動生成支撐地平線（Ground Line）。

## 1. 幾何幾何定義 (Geometric Constants)

根據 `腳.html` 的幾何結構與設計圖：
- **長軸半徑 (a)**: 25 單位 (Scale: $L_{leg} / 45$)
- **短軸半徑 (b)**: 13 單位 (Scale: $L_{leg} / 45$)
- **中心偏移**: 橢圓幾何中心相對於腿部底部安裝孔（$P_{bottom}$）的距離約為 $23 \times (L_{leg} / 45)$。

## 2. 接觸點（Point of Contact）計算

當腿部相對於垂直方向旋轉角度為 $\theta$ (代碼中為 `rot`)，且地面斜率為 $m$ 時，橢圓在特定方向上的高度位移 $h$ 可以精確求出。

### 核心公式
橢圓 $ (x/a)^2 + (y/b)^2 = 1 $ 旋轉 $\theta$ 後，與斜率為 $m$ 的切線之間的中心垂直距離 $h$ 為：

$$h = \sqrt{a^2(\sin\theta - m\cos\theta)^2 + b^2(\cos\theta + m\sin\theta)^2} / \sqrt{1 + m^2}$$


### 代碼實作 (`getEllipticFootPoint`)
1. **定位中心**: 定位橢圓在世界座標系中的中心 $(Cx, Cy)$。
2. **計算局部旋轉**: `rot = atan2(dy, dx) - PI/2`。
3. **計算位移**: 套用上述公式得到 $h$。
4. **輸出點位**: 回傳 `{ x: Cx, y: Cy - h }` (y 向上為正)。

## 3. 地平線搜索（Ground Plane Search）

模擬器使用「動態切點凸包」算法：
1. **接觸點映射**: 每一幀使用前一幀的緩衝斜率 $m_{prev}$ 預估每隻腳的 $P_{eff}$。
2. **窮舉搜尋**: 遍歷所有腳尖配對 $(P_i, P_j)$，建立直線 $y = mx + c$。
3. **凸邊驗證**: 若所有其他腳尖點 $k$ 皆滿足 $P_k.y \ge m P_k.x + c$，則該直線為「有效支撐面」。
4. **穩定支點**: 當機器發生傾斜（Tipping）時，物理引擎會自動鎖定最外側接觸點作為旋轉中心，確保動作自然。

## 4. 效能優化 (O(1) Complexity)

由於接觸點計算採用的是**代數閉式解 (Closed-form solution)** 而非迭代搜索，其運算開銷極小，僅涉及一次開平方與三角函數，完美解決了複雜幾何碰撞可能導致的幀率（FPS）下降問題。

## 5. 2026-03-18 精度調優更新內容

為了確保模擬器中的紅點（碰撞預測點）與 `腳.html` 的視覺零件完全重合，我們進行了以下修正：

### A. 常數標定修正 (Calibration)
- **原本偏移**: `centerDist = 23`
- **修正後**: **`centerDist = 28.54`**
- **依據**: 根據 `腳.html` 的 CAD 數據，支點（y=65）到足尖底部（y=106.54）的總長為 41.54。減去短軸半徑 $ry=13$ 後，得出中心偏移量為 28.54 單位。

### B. 接觸點鏡像算法升級 (Contact Symmetry)
原本的 `getEllipticFootPoint` 在處理傾斜（$rot \ne 0$）時，會誤將橢圓「頂部趨向點」的水平位移套用到「底部接觸點」，導致紅點在傾斜時會朝反方向偏移。

## 6. 核心算法深層解析 (`getEllipticFootPoint` 步驟 5-8)

本節詳細拆解代碼中如何利用「支持函數 (Support Function)」精確定位切點：

### 步驟 5：地面法線投影 (Ground Normal Projection)
在世界座標系中，斜率為 $m$ 的地面，其指向天空的法線向量為 $\vec{n} = (-m, 1)$。我們需要知道這個向量在橢圓「局部長軸 $\vec{u}$」與「局部短軸 $\vec{v}$」上的分量：
- **投影分量 $n_u$**: $\vec{n} \cdot \vec{u} = (-m, 1) \cdot (\cos\theta, \sin\theta) = \sin\theta - m\cos\theta$
- **投影分量 $n_v$**: $\vec{n} \cdot \vec{v} = (-m, 1) \cdot (-\sin\theta, \cos\theta) = \cos\theta + m\sin\theta$
這一步將「外部環境斜率」轉化為「相對於腳部姿態」的局部參數。

### 步驟 6：計算支持半徑 $h$ (Support Radius)
根據橢圓幾何性質，中心到法線方向切線的垂直距離 $h$（即支撐函數值）為：
$$h = \sqrt{a^2 n_u^2 + b^2 n_v^2}$$
這代表了橢圓在該特定方向上「伸展」的最遠距離。

### 步驟 7：定位局部切線點 (Relative Displacement)
在局部座標系中，法線向量為 $(n_u, n_v)$ 的切點坐標為 $(x_{loc}, y_{loc}) = (a^2 n_u / h, b^2 n_v / h)$。
代碼中透過旋轉矩陣將此位移轉換回世界座標系 $\Delta P_{top} = (dx_{rel\_top}, dy_{rel\_top})$。這組位移指向的是橢圓與地面平行、但在「上方」的切點。

### 步驟 8：對稱鏡像與碰撞點輸出 (Contact Point Output)
由於橢圓具有幾何中心對稱性，我們需要的地面接觸點正好是上方切點的**反向投影**：
$$\vec{P}_{cp} = \text{Center} - \Delta P_{top}$$
最終輸出的 $\{x, y\}$ 同時包含了足型弧度引起的垂直偏差與水平滑移。

## 7. 公式推導：垂直位移 $h$ 的來源

核心公式 $h = \sqrt{a^2(\sin\theta - m\cos\theta)^2 + b^2(\cos\theta + m\sin\theta)^2} / \sqrt{1 + m^2}$ 是透過**支持函數 (Support Function)** 理論推導而出的：

1.  **局部座標系**: 設橢圓方程為 $x_L^2/a^2 + y_L^2/b^2 = 1$。
2.  **法線向量**: 世界坐標地面法線為 $\vec{n} = (-m, 1)$。透過旋轉矩陣 $\mathbf{R}_{\theta}$ 將其投至橢圓軸向：
    - $n_u = \vec{n} \cdot (\cos\theta, \sin\theta) = \sin\theta - m\cos\theta$
    - $n_v = \vec{n} \cdot (-\sin\theta, \cos\theta) = \cos\theta + m\sin\theta$
3.  **支持函數定義**: 橢圓中心到法線向量 $\vec{w}$ 方向切線的垂直距離 $d$ 為：
    $$d = \frac{\sqrt{a^2 w_x^2 + b^2 w_y^2}}{|\vec{w}|}$$
    （註：這是由切線方程 $x_L x_0 / a^2 + y_L y_0 / b^2 = 1$ 與點到直線距離公式推導而來）
4.  **代入結果**: 將 $n_u, n_v$ 作為 $w_x, w_y$，並代入 $|\vec{n}| = \sqrt{m^2+1}$，即可得證。
